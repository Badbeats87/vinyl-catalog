

Admin flow:
- Define pricing strategy:
    - Buy price: admin-defined percentage of lowest/median/highest (admin-defined) discogs price. If discogs data not available -> admin-defined percentage of lowest/median/highest (admin defined) eBay price. Condition (Media + Sleeve): Must be able to define discounts based on condition.
    - Sell price: admin-defined percentage of of lowest/median/highest (admin-defined) discogs price. If discogs data not available -> admin-defined percentage of lowest/median/highest (admin defined) eBay price. Condition: Must be able to define discounts based on condition.
- Can search for records (RELEASES, NOT MASTER RELEASE):
    - Gets buy and sell price suggestions based on pricing strategy defined earlier.
    - Can add to inventory.
- Can go to inventory and define: condition, exact release, etc. 
- In inventory: can set record to live or draft. Gets automatic selling price suggestions based on pricing strategy. Live = For sale on the store front. Can adjust condition, and override price.
- When a seller offers a record, must be able to review, inspect details, etc.

Seller flow:
- visits website
- Searches for the record they want to sell
- Get results for MASTER RELEASE based on admin-defined pricing strategy.
- Can add record to submission. 
- Must add condition (Media + Sleeve) before submitting.
